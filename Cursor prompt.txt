## 6. SessionService (services/session_service.py)

Methods:

create_session(db, staff_id, ip_address=None, user_agent=None) → tuple[str, Session]
  - generate 64-byte raw token with secrets.token_hex(64)
  - store SHA-256 hash in token_hash column
  - set expires_at = now + 24 hours
  - return (raw_token, session_object) — raw token returned ONCE only, never stored

get_session_by_token(db, raw_token) → Session | None
  - hash the raw token, look up by token_hash
  - return None if not found, revoked=True, or expires_at < now

revoke_session(db, session_id) → None
  - sets revoked=True

revoke_all_for_staff(db, staff_id) → int
  - sets revoked=True on all active sessions for that staff member
  - returns count revoked

## 7. Discord OAuth Service (services/discord_oauth.py)

Methods — all are pure HTTP calls, no DB access:

get_auth_url(state) → str
  - build Discord OAuth2 URL with:
    - client_id from settings.discord_client_id
    - redirect_uri from settings.discord_redirect_uri (add this field to config/settings.py with default "http://localhost:3000/auth/callback")
    - scope: "identify"
    - response_type: "code"
    - state param included
  - return full URL string

exchange_code(code) → dict
  - POST to https://discord.com/api/oauth2/token
  - fields: grant_type=authorization_code, code=code, redirect_uri from settings
  - auth: client_id + client_secret (basic auth style as per Discord docs — send as form data with client_id and client_secret fields)
  - return parsed JSON response (contains access_token, token_type, etc.)
  - raise HTTPException(502) if Discord returns non-200

get_user(access_token) → dict
  - GET https://discord.com/api/users/@me
  - Authorization: Bearer {access_token}
  - return parsed JSON (contains id, username, discriminator, avatar)
  - raise HTTPException(502) if non-200

Use httpx.AsyncClient for all HTTP calls.
